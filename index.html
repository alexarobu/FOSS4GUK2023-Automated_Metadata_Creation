Q-FAIR is all about making your data high-quality, findable, accessible, interoperable and reusable. This is best (or most easily) achieved if your metadata is also Q-FAIR, and also best achieved using open standards and formats where possible.

Metadata Crawler open source plugin for Talend Open Studio (aka Crawler)

Crawler looks for spatial data in file systems and databases.

It creates a metadata record for each data source it discovers, and a Feature Catalog record for vector data sources.

We generate a spreadsheet with a row per record for data owners to complete the manual fields.

How do we automate these? üßë‚Äçüíª

"industrial strength open source NLP in Python"

Easy to pick up, actively developed and used, and well documented. We looked at other approaches but they weren't such a good fit.

Geographic Keywords üåç

NER model trained on OS Open Names

Pattern recognition model (postcodes)

This was the trickiest element to handle. Geoparsing is definitely a "Thing", but generally not very fine-grained. We needed it to recognise UK roads, any sort of settlement, mountains, rivers, and postcodes

We built two custom pipelines- one that used NER trained on the OS Open Names dataset, and one that uses pattern recognition to pick up postcodes.

This approach was best for processing efficiency and can be run on a normal laptop

Non-Geographic Keywords üè∑Ô∏è

There are loads of options for this and we're not sure yet which approach gives consistently the best results, so we're currently running a few of them

INSPIRE Keywords and Topic Categories üîñ

Text Classification

These are controlled text fields so we're using text classification to pick the keywords/categories that best match the data.

Everything Else

For this we're using the OpenAI API to generate the abstract, the human-readable title, and to pick the most popular results from all the other modules. We include a confidence score to help our clients prioritize the workload for large data holdings
